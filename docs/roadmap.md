## Ближайшие улучшения

- [x] **Push-to-talk режим** — удерживаешь хоткей — идёт запись, отпускаешь — распознавание
- [ ] **Контекстные команды** — разные наборы команд для разных приложений (Terminal, VS Code, браузер)
- [ ] **Голосовая обратная связь** — озвучка подтверждений через TTS

- [ ] **Пограничные случаи trailing-space** — вставка пробела реализована как trailing space (пробел после каждого фрагмента, `_prepare_text` в `engine.py`, исключения `_NO_TRAILING_SPACE_AFTER = {"\n", " "}`). Известные краевые кейсы, где пробел нежелателен, если фрагмент приходит отдельно (сейчас редки, Whisper обычно отдаёт целиком — ловить только если начнёт мешать):
  - составные слова через дефис: `интернет-` + `магазин` → `интернет- магазин`;
  - пути/дроби через слеш: `and/` + `или` → `and/ или`;
  - открывающие парные символы: `(`, `«`, `„` в конце фрагмента → пробел после открывающей скобки/кавычки (`( текст`).

## Реализовано

- [x] **Вставка через буфер по умолчанию, без дублей и без следов** — `SHEPTUN_USE_CLIPBOARD=true` по умолчанию: Cmd+V атомарен и убирает дублирование символов в терминале VS Code / Electron (событийный ввод `_send_via_events` негерметичен — правился таймингом/флагами без полного результата). Буфер помечается `org.nspasteboard.ConcealedType` (не только `TransientType`), поэтому голосовые вставки не попадают в историю clipboard-менеджеров, включая CopyClip (blacklist приложения в менеджере не работает: агент не frontmost, вставка атрибутируется целевому приложению). Перед вставкой снимается снапшот **всех** типов буфера (`NSPasteboardItem`) и восстанавливается после — картинки/файлы/RTF больше не теряются (`_snapshot_clipboard`/`_restore_clipboard` в `keyboard.py`)
- [x] **LLM-анализ логов → автозамены** — `sheptun analyze-replacements`: анализ лога распознавания через Anthropic SDK, предложение правил автозамены с контекстом (±N строк), инкрементально по чекпоинту, verify-стадия (критик перепроверяет кандидатов). Skill `review-replacements` для ревью результата
- [x] **Пользовательский словарь автозамен** — `replacements.yaml`, применяется после распознавания по границам слов (см. также «Средняя перспектива» и «Персонализация»)

## Universal Control — иконка PTT на remote

Проблема: в PTT режиме при курсоре на remote машине (Universal Control) иконка на хосте не становится залитой во время записи.

Корень: нажатие PTT хоткея заставляет UC отдать фокус хосту раньше чем `_on_ptt_start` успевает сработать. Из-за этого `_uc_active` уже `False` к моменту установки иконки.

Что пробовали:
- `NSWorkspaceDidActivateApplicationNotification` вместо polling — работает для idle/toggle, но нажатие хоткея меняет frontmost app раньше PTT start
- Debounce 300ms для сброса `_uc_active` — UC теряет фокус за ~1 секунду до PTT, debounce не успевает
- Координаты мыши (`NSEvent.mouseLocation`) вместо frontmostApplication — UC не расширяет координатное пространство macOS, cursor остаётся в пределах локального экрана

Идеи для решения:
- Сохранять timestamp последней UC активности, в `_on_ptt_start` считать UC активным если прошло менее ~1.5 сек (эвристика, риск false positive)
- Искать другой системный API для определения UC состояния (IOKit, CoreGraphics, приватные фреймворки Apple)
- Подавлять PTT хоткей в pynput (suppress=True) чтобы клавиша не попадала в систему и не меняла фокус
- Принять как известное ограничение

## Долгосрочно

- [ ] **Диктовка с пунктуацией** — "привет запятая как дела вопрос" → "Привет, как дела?"
- [ ] **LLM-enhancement (опциональный режим)** — прогон транскрипта через локальную LLM для причёсывания: пунктуация, капитализация, грамматика, слова-паразиты, устные самокоррекции («нет стоп»), словесная разметка. Как «AI Enhancement» у VoiceInk. Исследовано (см. `docs/llm-enhancement-research.md`): рабочая формула — reasoning off + few-shot правила + компактный словарь терминов в промпте, `replacements.yaml` детерминированным слоем ДО LLM. Блокер на текущем железе: LM Studio-движки требуют macOS 14+/Metal 3.1 (у нас 13.7) — обходится своим `mlx-lm` в venv (как MLX-Whisper) или откатом рантайма. Только опциональный режим (задержка ~1-3 сек против «мгновенно»)
- [ ] **Мультиязычность** — переключение языков голосом, смешанная речь

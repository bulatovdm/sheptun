# Ансамблирование ASR — исследование

*Deep-research 2026-07-05 (109 агентов, 26 источников, 25 фактов проверено adversarial: 24 подтверждено, 1 опровергнут). Вопрос: стоит ли комбинировать несколько ASR-движков для нашего кейса — русская диктовка с англо-терминами, реалтайм, локально на M2.*

## Вердикт: полноценный ансамбль двух ASR — overkill. Одна ветка достойна внимания — каскад Whisper N-best → LLM.

Твой текущий путь (одиночный Whisper turbo + `replacements.yaml` + опциональный LLM-enhancement) — обоснованно лучший. Единственный ансамбле-смежный апгрейд с сильной доказательной базой — LLM-коррекция над **N-best** гипотезами Whisper (не над 1-best).

## 1. Параллельный ансамбль (ROVER / MBR / CNC) — работает, но не для нас
- ROVER (Fiscus 1997), confusion-network combination, MBR-комбинация (Kaldi; Povey 2011; CAPIO 2017) — реальные production-grade методы: несколько движков → выравнивание гипотез в сеть → голосование по словам.
- **Выигрыш скромный:** ~0.6–1.9 абс. пункта WER над лучшим одиночным (англ. бенчмарки); чистый ROVER ~0.9. Раздутая цифра «16% relative» — **опровергнута** (0-3).
- **Не подходит нам:** ROVER помогает только когда движки СОПОСТАВИМЫ по точности и разнообразны (IEEE 6727393). У нас Whisper силён, а GigaAM/Parakeet на англо-терминах — мусор (транслитерация, проверено). Голосование «хороший vs мусор» ухудшит, а не улучшит.

## 2. Каскад с LLM (N-best → генеративная коррекция) — СИЛЬНЕЙШАЯ ветка, релевантна нам
- HyPoradise (NeurIPS 2023), Whispering LLaMA (37.66% rel. WERR), RobustGER (до 53.9%), GenSEC H2T (11.82%→8.33% WER), Apple delayed fusion.
- Ключевое: LLM **исправляет даже токены, отсутствующие во всех гипотезах** → превосходит потолок обычного rescoring.
- **Прямо про наш кейс:** GFD доказанно улучшает code-switching (мандарин-английский), выигрыш авторы приписывают тому, что «LLM исправляет встроенную английскую терминологию» — механически = русский + англо-термины.
- Оговорки: сильнейшие результаты — на LLM, ДООБУЧЕННОЙ на парах гипотеза→эталон (LoRA); zero-shot слабее. Все цифры — англ./кит., НИ ОДНОЙ на русском (перенос, не измерение).
- **Практический вывод для нас:** отдавать LLM не 1-best, а **N-best** Whisper (mlx-whisper это умеет) — больше сигнала, заметно лучше коррекция. Это апгрейд текущего enhancement-плана, не второй ASR.

## 3. Доменная специализация (русский от одного + латиница от другого) — концепт есть, дорогой
- Валидно (arXiv 2211.01046): два моноязычных ASR → ОБУЧАЕМЫЙ нейронный fusion-модуль (BELM), двухстадийно.
- **Не для нас:** тренируемый модуль, только мандарин-английский, без русского/реалтайма/Mac. Не plug-and-play «слить два вывода».

## 4. Практическая цена локально — НЕ измерена никем
- Ни один источник не замерил латентность/память 2+ ASR одновременно на потребительском Apple Silicon в реалтайме → главный вопрос «окупается ли» фактами не подтверждён.
- Побочно: WhisperKit (Argmax, ICML 2025) — Whisper large-v3-turbo на ANE, 0.46с латентность, 2.2% WER, M2 Ultra ~42× RT. Одиночный Whisper на Mac уже быстрый.

## Открытые вопросы (если возвращаться к теме)
- Замерить реальную латентность/память двух движков (Whisper + GigaAM/T-one) на M2 через MLX/CoreML — окупается ли для sub-second диктовки.
- Есть ли измерения GER/H2T/GFD на РУССКОМ (все данные — англ./кит.).
- Захватывает ли лёгкий LLM-каскад над ОДНИМ Whisper 1-best/N-best большую часть выгоды code-switching, делая настоящий мульти-ASR ненужным (гипотеза: да).
- Можно ли дёшево приблизить доменную специализацию: Whisper на всё + точечная коррекция англо-терминных спанов словарём/LLM, без второго ASR и обучаемого BELM.

# Бенчмарк корректоров текста: JamSpell vs SAGE

*Прогон от 2 июля 2026 на реальных логах Sheptun. Инструмент: `benchmarks/` (`python -m benchmarks run`).*

## Вывод

**JamSpell не подходит категорически — он систематически ломает англотермины.
SAGE на порядок безопаснее, но НЕ абсолютно безопасен** — на редких плотно-англоязычных
фразах тоже галлюцинирует. Для прода нужен либо контроль этих кейсов, либо связка с
существующим `replacements.yaml`. Подтверждено на 5000 реальных строк.

## Результаты — 5000 строк из логов (`--source log`, greedy+batch16)

| Корректор | Изменено | **Повреждено** | Потеряно lat/термы | мс/строку |
|---|---|---|---|---|
| noop (baseline) | 0% | 0% | 0/0 | 0 |
| **JamSpell** (ru_small, 81 МБ) | 22% | **1% (62 строки)** | 0/**62** | 1 |
| **SAGE** (fredt5-distilled-95m, 0.38 ГБ) | 44% | **0.5% (23 строки)** | **18/8** | 58 |

*(выборка: 826 строк с латиницей, 113 с транслит-англотерминами)*

### JamSpell — 62 повреждения, все однотипные
Систематически `коммит`→`кормит` на каждой git-фразе. Приговор подтверждён на объёме:
```
было : Давай сделай коммит и пуш.
стало: Давай сделай кормит и пуш.
было : Да, делаем коммит, ссылка на задачу та же.
стало: Да, делаем кормит, ссылка на задачу та же.
```

### SAGE — 23 повреждения, ДВА разных типа
**Безобидные / канонизация** (не поломка смысла):
```
dbdump → db-dump          relations → relationship
```
**Реальные поломки** (критично для голосовых команд):
```
skip health check → skip-to-chick     ← команда уничтожена
third party       → Trice Party        ← выдумал
bingconsole       → Bingcomsole        деплой → дэплой
```
Вывод: SAGE безопасен на смешанной речи с ОТДЕЛЬНЫМИ англицизмами, но на фразах,
где латиницы много подряд, начинает «переводить» её в фонетический бред. Таких — единицы
на тысячи, но каждая ломает команду.

## На 150 строках (для сравнения масштаба)

| Корректор | Изменено | Повреждено | мс/строку (beam=3) |
|---|---|---|---|
| JamSpell | 17% | 3% (4) | 1 |
| SAGE | 44% | 0% (0) | 351 |

На малой выборке SAGE выглядел идеально (0 повреждений) — большая выборка вскрыла редкие поломки.
Урок: гонять на объёме, а не на 150 строках.

## Результаты — 100 пар из replacements.yaml (`--source replacements`, с эталоном)

| Корректор | Изменено | Точность (exact-match) | мс/строку |
|---|---|---|---|
| noop | 0% | 1% | 0 |
| JamSpell | 41% | 8% | 1 |
| SAGE | 100% | 0% | 140 |

**Важный нюанс:** низкая точность SAGE здесь НЕ означает, что он плох. `replacements.yaml` содержит специфичные **фонетические ASR-искажения одиночных слов** («комит»→«коммит»), которые SAGE не обучен чинить — он про орфографию/пунктуацию/регистр, а не про ASR-каши. Плюс SAGE добавляет пунктуацию/регистр, из-за чего строгий exact-match падает.

## Ключевой вывод для архитектуры

**SAGE и `replacements.yaml` — не конкуренты, а взаимодополняющие стадии:**
- `replacements.yaml` (уже есть) — чинит доменные ASR-искажения и переводит транслит→латиницу (`докер`→`docker`).
- **SAGE** (кандидат на добавление) — причёсывает пунктуацию, регистр, общую орфографию; англотермины
  в основном бережёт, НО на плотно-англоязычных фразах может «перевести» латиницу в бред (0.5% случаев).
  Порядок `replacements → SAGE` частично страхует: replacements сперва переведёт транслит в латиницу,
  но SAGE может испортить уже её — связку надо измерить (см. открытые вопросы).

Оптимальный порядок в пайплайне: `галлюцинации → replacements → SAGE → форматирование → команды`. SAGE встаёт как новый корректор в `spelling.py` (архитектура уже готова — `SHEPTUN_SPELL_CORRECTION`), рядом с существующим T5.

## Скорость SAGE — оптимизация (реализовано)

Настройки в `benchmarks/correctors/sage.py` (env): `SHEPTUN_BENCH_SAGE_BEAMS`,
`_BATCH`, `_DEVICE`. Замер на CPU (Apple Silicon, macOS 13.7):

| Конфиг | мс/строку | Ускорение |
|---|---|---|
| beam=3, batch=1 (было) | 368 | 1× |
| greedy (beam=1) | 150 | 2.5× |
| greedy + batch=8 | 89 | 4.1× |
| **greedy + batch=16 (дефолт)** | **51** | **7.2×** |

Дефолт — greedy+batch16. `BEAMS=3` для максимальной точности (beam чуть точнее на edge-кейсах
вроде `контейнер` vs greedy `конвертер`, но англотермины greedy сохраняет).

**⚠️ MPS (Apple GPU) НЕ работает на этой машине:** torch 2.9 требует **macOS 14.0+**, у
пользователя **13.7.8**. Всё идёт на CPU. `device=auto` сам подхватит MPS после апгрейда
ОС до Sonoma — тогда ускорение будет кратно больше. См. память `reference_mps_macos_requirement`.

## Как воспроизвести

```bash
pip install -e ".[bench]"
export SHEPTUN_BENCH_JAMSPELL_MODEL=/path/to/ru_small.bin   # см. benchmarks/README.md
python -m benchmarks run --correctors noop,jamspell,sage --source log --count 5000
python -m benchmarks run --correctors noop,jamspell,sage --source replacements --count 100
# точность vs скорость: SHEPTUN_BENCH_SAGE_BEAMS=3 python -m benchmarks run ...
```

## Открытые вопросы

2. Как SAGE ведёт себя ПОСЛЕ replacements.yaml (на уже частично исправленном тексте) — измерить связку, а не по отдельности.
3. Даёт ли SAGE-95m прирост читаемости, оправдывающий 0.38 ГБ + латентность, против «просто replacements + голосовые команды пунктуации» (текущее состояние)?

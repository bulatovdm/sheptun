# Проверка на чувствительные данные

Комментарии в `replacements.yaml` содержат фрагменты распознанной речи (контекст ошибок).
Перед тем как что-либо оставлять или коммитить, проверь, что туда не утекли секреты/личные данные.

## Сканы

Секреты (ключи, токены, пароли, URL с кредами):
```bash
grep -inE "api[_-]?key|secret|token|password|passwd|bearer|sk-[a-z0-9]|ghp_|xox|-----BEGIN|://[^ ]*:[^ ]*@" src/sheptun/config/replacements.yaml || echo "не найдено"
```

Email, домены, IP:
```bash
grep -inE "[a-z0-9._%+-]+@[a-z0-9.-]+|[0-9]{1,3}(\.[0-9]{1,3}){3}|https?://" src/sheptun/config/replacements.yaml || echo "не найдено"
```

Самые длинные комментарии (проверить на утечку целых фраз):
```bash
awk -F'#' 'NF>1{print length($2), $0}' src/sheptun/config/replacements.yaml | sort -rn | head -5 | cut -d' ' -f2-
```

## Что делать при находке
- Название технологии/сервиса (WireGuard, MODX, Bitrix и т.п.) — НЕ чувствительно, это норма.
- Реальный секрет/личные данные — пометь правило как WRONG, вычисти из комментария/удали правило,
  сообщи пользователю. Не коммить, пока не убрано.
